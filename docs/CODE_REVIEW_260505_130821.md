## 缺陷清单

### 严重 (CRITICAL)

#### C1. KVDB 迭代器 uint16_t 截断
- **文件**: `src/rocketdb_kvdb.c:2790,2796,2800`
- **描述**: `it->offset` 类型为 `uint32_t`，但被 `(uint16_t)` 强制转换截断。`cur_off` 变量声明为 `uint16_t`。当 `sector_size > 65535` 时（如 256KB NOR 扇区），`(uint16_t)ss = 0`，导致迭代器无限循环。
- **影响**: 扇区大小超过 64KB 时死循环，缓冲区溢出风险。
- **修复**: `cur_off` 改为 `uint32_t`，移除 `(uint16_t)` 强制转换。

### 高 (HIGH)

#### H1. set() 提交失败未推进 write_off
- **文件**: `src/rocketdb_kvdb.c:2327-2331`
- **描述**: 提交字节写入失败时直接返回 RDB_ERR_FLASH，`write_off` 未推进（推进代码在 2335 行，return 之后）。下次写操作会复用同一地址，违反 NOR Flash 单次写入约束。
- **影响**: 提交失败后的记录区域无法恢复，后续写入可能产生数据损坏。
- **修复**: 失败后推进 write_off 并标记记录为 DEAD。

#### H2. iter_gen++ 在 fl_unlock() 之后执行
- **文件**: `src/rocketdb_kvdb.c:2359-2360`
- **描述**: `iter_gen++` 在 `fl_unlock(db)` 之后执行，与其他操作（delete/gc）在锁内更新 iter_gen 不一致。多线程环境下产生竞态窗口。
- **影响**: 迭代器过期检测窗口扩大。
- **修复**: 将 `iter_gen++` 移到 `fl_unlock()` 之前。

#### H3. format() 静态缓冲区违反零分配约束
- **文件**: `src/rocketdb_kvdb.c:2072`, `src/rocketdb_tsdb.c:1066`
- **描述**: `static uint32_t saved_ec[RDB_MAX_SECTORS]` 使用 BSS 段内存，违反 API 文档声明的"所有缓冲区由调用者提供"。
- **影响**: 不可重入，多线程不安全，增加 BSS 占用。
- **修复**: 改为栈上 VLA（scnt ≤ 255，栈空间安全）。

#### H4. FAULT_TYPE_READ_FAIL 和 FAULT_TYPE_BIT_FLIP 未测试
- **文件**: `test/sim/sim_fault.h:34,39`, `test/sim/test_fault_injection.c`
- **描述**: 两种故障类型已在枚举中定义并在报告中打印计数，但从未被任何测试用例使用。
- **影响**: 读失败和位翻转路径完全未测试。
- **修复**: 在 test_fault_injection.c 中添加对应测试用例。

#### H5. GC/Rotation 压力目标从 100 降至 13
- **文件**: `test/sim/test_kvdb_stress.c:103`, `test/sim/test_tsdb_stress.c:95`, `test/sim/test_integration.c:144-145`
- **描述**: 测试目标常量 GC_STRESS_TARGET/ROT_TARGET/CYCLE_GC_TARGET/CYCLE_ROT_TARGET 均为 13，但测试名称、描述和 test_plan.md 均声称 >=100。
- **影响**: 磨损均衡验证严重不足，仅约 1 次完整扇区遍历。
- **修复**: 恢复目标至 100。

#### H6. 随机种子未记录到跟踪日志
- **文件**: `test/sim/test_kvdb_stress.c:374`, `test/sim/test_tsdb_stress.c:300`, `test/sim/test_integration.c:130`
- **描述**: 可复现测试使用的 PRNG 种子是编译期常量，从未输出到跟踪日志中。
- **影响**: 测试结果无法事后验证复现条件。
- **修复**: 在测试开始时通过 trace_event() 输出种子值。

#### H7. fault_import_rules() 为空桩
- **文件**: `test/sim/sim_fault.c:402-410`
- **描述**: 函数标记为 TODO，"accepts any input" 静默丢弃。导出/导入往返被破坏。
- **影响**: 故障规则无法从字符串配置。
- **修复**: 实现基本规则解析。

#### H8. sim_runner.c 和 sim_vectors.c 为死代码
- **文件**: `test/sim/sim_runner.c`, `test/sim/sim_vectors.c`
- **描述**: 文件存在但从未被任何构建系统编译。SpecKit 文档仍引用它们。
- **影响**: 构建文档与实际不一致。
- **修复**: 清理或集成到构建系统。

### 中 (MEDIUM)

#### M1. 构建脚本 "build" 动作未实现
- **文件**: `project/build/run_all_tests.bat:39-40`（原路径）
- **描述**: ACTION=build 时仍执行编译+运行，未跳过测试执行。
- **修复**: 已在 Phase 1 整合至 build/ 时修正。

#### M2. sim_flash.c 单字节写入绕过对齐检查
- **文件**: `test/sim/sim_flash.c:63-65`
- **描述**: `len == 1` 时跳过写入粒度对齐检查。当 write_gran > 1 时，单字节写入应被拒绝。
- **影响**: 模拟器对单字节写入未强制执行对齐约束。
- **修复**: 移除 len==1 的特殊处理。

#### M3. wear_heatmap 测试断言永真
- **文件**: `test/sim/test_integration.c:435-436`
- **描述**: `TEST_ASSERT_GE(kv_max, kv_min)` — max 总是 >= min，无法验证磨损均衡分布。
- **影响**: 测试名义上验证磨损均衡，实际无有效断言。
- **修复**: 添加分布均匀性断言（如 max/min <= 阈值）。

#### M4. 共享 PRNG 状态跨测试泄漏
- **文件**: `test/sim/test_integration.c:130,211`
- **描述**: 全局 PRNG 状态在测试用例间共享，未在测试间重置。
- **影响**: 测试执行顺序影响结果。
- **修复**: 每个测试用独立种子重置 PRNG。

#### M5. ts_rotation_stress 可能读取栈垃圾
- **文件**: `test/sim/test_tsdb_stress.c:103,109`
- **描述**: `data[512]` 固定缓冲区，但随机尺寸 `dsz` 可能 > 512。
- **影响**: 罕见情况读取栈垃圾。
- **修复**: 将缓冲区扩展到最大尺寸或夹紧 dsz。

#### M6. GC 评分循环无 yield()
- **文件**: `src/rocketdb_kvdb.c:1563-1678`
- **描述**: GC 的 Phase 1-4 循环中无 yield 调用，对于大量扇区可能触发看门狗超时。
- **影响**: 协作式 RTOS 中可能任务饥饿。
- **修复**: 在长时间循环中插入 yield。

#### M7. 缺失 const 限定符
- **文件**: `src/rocketdb.h:936,947,1154,956,1175`
- **描述**: 只读查询函数（space_info, wear_info, get_stats）的 `db` 参数未用 const。
- **影响**: const 正确性缺失。
- **修复**: 添加 const 限定。

### 低 (LOW)

#### L1. Doxyfile 版本号为 1.0.0
- **文件**: `Doxyfile:6`
- **描述**: 硬编码版本 1.0.0，实际为 1.1.2。

- **文件**: `CONTRIBUTING.md` — 断链(design.md, build_and_run.sh)
- **文件**: `test/perf/README.md` — 版本号 v0.0.2→v1.1.2

#### L3. examples/ 目录为空
- **文件**: `examples/`, `CMakeLists.txt:98-99`
- **描述**: CMakeLists.txt 引用的示例文件不存在。
- **修复**: 添加示例文件或更新 CMake。

#### L4. TSDB CRC 双重读取
- **文件**: `src/rocketdb_tsdb.c:1740-1753,1848-1877`
- **描述**: get_latest/get_oldest 中数据从 Flash 读取两次（一次用于 CRC，一次用于输出）。
- **影响**: 2x Flash 读取带宽。
- **修复**: 合并 CRC 计算和数据拷贝为单次读取。

#### L5. SimFlash before-image 缓冲区固定 512 字节
- **文件**: `test/sim/sim_flash.c:77-78`
- **描述**: 跟踪日志的 before-image 缓冲区硬编码 512 字节，大于此的写入被静默截断。
- **修复**: 增大缓冲区或使用动态分配。
