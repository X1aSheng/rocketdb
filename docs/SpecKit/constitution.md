# Constitution（宪法）

## 目的
定义 RocketDB 模块的不可变核心目标、边界与根本约束。

## 核心目标

### KVDB（键值存储引擎）
1. 为少量 key（10~1000）的高频率更新提供高效存储。
2. 自动垃圾回收（GC）机制，消除碎片化，维护稳定的可用空间。
3. 四阶段评分 GC 策略，结合静态磨损均衡（Phase 4），保障全盘扇区寿命均衡。
4. 掉电一致性保证：任意时刻断电，下次初始化恢复到一致状态。

### TSDB（时序存储引擎）
1. 为追加式时序数据（传感器采样、告警日志）提供环形缓冲存储。
2. 自动时间戳 epoch 管理，防止时间回绕导致的数据查询错误。
3. 环形 rotation 机制，扇区满时自动切换至下一扇区，最旧数据自动淘汰。
4. 原生支持时间范围查询（from_ts ≤ timestamp ≤ to_ts）。

## 根本约束

### 零动态内存
- **无堆分配**：全部缓冲区由调用者提供或栈上分配。
- **防碎片**：杜绝内存碎片，适应嵌入式系统限制。
- **后果**：API 设计中，meta_buf、iter_buf 等均为输入参数。

### NOR Flash 安全性
- **1→0 原理**：全部写操作严格遵循"仅可将 1 清为 0"，无原地覆盖。
- **二次写保护**：不依赖 Flash 硬件的区间锁（某些芯片不支持），而是算法上保证。
- **掉电恢复**：采用"先写新数据后删旧"策略，任何路径不以丢失已有数据为代价。

### 掉电一致性与确定性 GC
- **两阶段写**：记录先以 WRITING (0xFF) 状态写入，CRC 校验通过后提交为 VALID (0xFE)。
- **GC 前可行性校验**：迁移前完整扫描判断目标可用空间，不会出现中途无处可写。
- **防死锁兜底**：初始化时 Phase 4 强制恢复 gc_reserve + 1 个擦除扇区。

### 结构体对齐与硬件兼容性
- **自然对齐**：所有 On-Flash 结构体 32 位字段精确对齐到 4 字节边界。
- **HardFault 防护**：Cortex-M0 等不支持非对齐访问的 MCU 上，可直接将 flash 读缓冲强制转换为结构体指针。
- **位填充**：padding 字段显式定义，初值为 0xFF（未使用状态）。

### 数据安全优先
- **不扔数据**：任何 GC、rotation、recovery 路径均不以丢失有效数据为交换。
- **CRC 校验**：所有 On-Flash 数据记录含 CRC16，损坏自动标记或隔离。
- **sector 隔离**：单个扇区损坏不影响其他扇区的读写。

### 实时性保障
- **yield 回调**：GC、rotation、扫描期间定期调用 yield() 喂狗、让出 CPU。
- **主动监控**：garbge_bytes、live_bytes 的实时维护，避免大型集中扫描。

## 非目标

### 不追求的能力
1. **向后兼容**：版本升级不需考虑旧格式数据迁移。
2. **外部库依赖**：除 CRC16、hash16 外，不引入其他库。
3. **集群分布式**：仅面向单 MCU + 单 Flash 场景。
4. **ACID 事务**：仅支持 record 级别的原子性，无全局事务。
5. **加密/认证**：数据不加密，不包含身份认证机制。

## 约束条件

### 编译时配置
- API 兼容 C99 标准。
- 禁止使用 `bool` 类型（用 `uint8_t` 代替）。
- 所有操作返回 `rdb_err_t` 错误码。

### 硬件约束
- **Flash 类型**：SPI NOR Flash（W25Q 系列及兼容型号）。
- **MCU 范围**：Cortex-M0 / M3 / M4 / M7 及等效处理能力。
- **扇区大小**：4KB（可扩展至 8KB、16KB，但编译时固定）。
- **分区大小**：单分区 ≤ 1MB；KVDB 分区数 ∈ [3, 255]；TSDB 分区数 ∈ [2, 255]。

### 典型工作场景

### KVDB 工况
```
Key 数量：10~1000（典型 20~50）
Value 大小：4~4095 字节（典型 32~256 字节）
操作模式：少量 key 频繁更新
GC 触发：自动（空间不足）或手动
稳态 GC：直接擦除 zero-live 扇区，开销最小
```

**示例场景**（来自 design.md）：
```
20 key, avg rec_sz=52B, 8 扇区 (32KB):
  存活数据 = 1040B（MAX_LIVE 的 3.6%）
  稳态 GC = 直接擦除 zero-live 扇区，开销最小
```

### TSDB 工况
```
单条数据大小：8~128 字节（典型）；支持接近单扇区容量的大记录
采样频率：1~1000 Hz（可配）
保留周期：可配，环形覆盖最旧数据
Query：时间范围查询 [from_ts, to_ts]，支持跨扇区查询
```

### 测试基准配置（来自 test_request.txt）

**硬件平台**：W25Q128 (16MB NOR Flash)

**分区配置**：
```
KVDB1: 32KB (8 扇区 × 4KB)
KVDB2: 32KB (8 扇区 × 4KB)
TSDB1: 32KB (8 扇区 × 4KB)
TSDB2: 32KB (8 扇区 × 4KB)
```

**GC 压力测试要求**：
- 每个分区至少触发 100 次 GC (KVDB) 或 rotation (TSDB)
- 验证不同写入/删除模式下的 GC 行为
- 确保全盘扇区寿命均衡（erase_cnt 差异 ≤ RDB_GC_WEAR_THRESHOLD）

**数据分布测试要求**（来自 test_plan.md 4.2.1）：
- 混合不定长 KEY 和 VALUE
- Key 长度：50% 短 (1-8B)，30% 中等 (9-24B)，15% 长 (25-31B)，5% 最大 (32B)
- Value 长度：40% 小 (0-32B)，40% 中 (33-256B)，15% 大 (257-1024B)，5% 最大
- 操作混合：60% insert，25% update，10% get，5% delete

**可靠性测试要求**：
- 掉电注入：在 header/data/commit 关键步骤模拟断电
- CRC 损坏注入：验证损坏数据隔离与恢复
- 序列回绕：KVDB write_seq 接近 0xFFFFFFFF 时的恢复
- Epoch 回绕：TSDB 时间戳回绕的正确处理

## 兼容性与可移植性

### 编译器支持
- **GCC / Clang**：可使用内建加速（popcount、ctz、clz）。
- **其他编译器**：回退为软件实现，功能不变。

### 平台支持
- **字节序**：假定小端序（little-endian）。
- **浮点数**：不使用浮点，所有时序计算采用整数。
- **线程安全**：通过 flash_ops_t 中的 lock/unlock 回调实现。
